# Educational RAG Assistant: Система генерации ответов для образовательных материалов

Проект посвящен разработке интеллектуального ассистента, способного обрабатывать сложные математические и технические учебники для генерации точных ответов на вопросы пользователя.

## 🚀 Основные возможности
- **Продвинутый парсинг (VLM-based):** Извлечение не только текста, но и сложных математических формул и изображений из PDF с использованием Vision Language Models.
- **Умный чанкинг:** Семантическое разделение текста с сохранением контекста формул и графиков.
- **Гибридный поиск:** Комбинирование векторного (semantic) и полнотекстового (keyword) поиска для работы с узкоспециализированной терминологией.
- **RAG-пайплайн:** Генерация ответов на базе извлеченного контекста с использованием современных LLM.

## 🛠 Технологический стек
- **Язык:** Python
- **AI/ML:** LLM, VLM для OCR, Embedding models (тестирование различных вариантов)
- **Data & Search**, Hybrid Search 
- **Инфраструктура:** API (FastAPI), Docker для контейнеризации

## 🔍 Исследования 

### Парсинг сложных образовательных материалов
Одной из ключевых задач проекта стало качественное извлечение данных из PDF-учебников, содержащих большое количество математических формул и схем.

**Протестированные подходы:**
* **Стандартные парсеры (PyMuPDF, PDFMiner):** Теряют структуру формул и игнорируют вложенные изображения.
* **Специализированные SOTA-инструменты (Marker, Nougat, MineU):** Наблюдались галлюцинации в сложных формулах и проблемы с парсингом русскоязычного текста. Наиболее удовлетворительный результат показал marker, который хорошо разбивает данные на текст/формулы/изображения и допускает не слишком много ошибок в тексте.

**Текущее решение:**
В данный момент я разрабатываю пайплайн постобработки на основе **VLM** для визуального анализа страниц. Это позволяет:
1. Сохранять пространственную структуру документа.
2. Корректно интерпретировать сложные формулы в контексте текста.
3. Автоматически описывать изображения для их последующей индексации в RAG.

Важно создать правильную фильтрацию неправильно распознанных данных, чтобы не постобрабатывать через VLM страницы полность.
